--- title: 信息管理与信息系统专业学习札记 date: 2026-07-14 tags: [信管, 学习总结, 博客搭建] --- # 信息管理与信息系统专业学习札记 **作者：吴睿轩** ## 关于我 我是吴睿轩，北京信息科技大学信息管理与信息系统专业的一名学生。 信息管理与信息系统（简称"信管"）是一个交叉学科，融合了管理学、计算机科学和系统科学的知识。选择这个专业，是因为我相信在数字化时代，能够理解"技术"与"管理"两种语言的人，会有更广阔的发展空间。 ## 一、为什么选择信管 高考填报志愿时，我面临一个选择：是学纯技术类的计算机，还是偏管理的工商管理？后来了解到信管这个专业，发现它正好处在两者之间。 信管专业要学的东西很杂，既包括管理学原理、经济学、运筹学等管理类课程，也包括程序设计、数据库、数据结构等计算机类课程，还有系统分析与设计这样整合性的课程。有人说这个专业"什么都学，什么都不精"，但我更愿意换一个角度看——它提供了一个跨学科的视野。 在信息系统中，纯粹的技术实现往往不是最难的，最难的是理解业务需求、设计合理的系统架构、协调各方资源。这些能力，恰好是信管专业的培养方向。 ## 二、技术学习路线 基于专业要求和个人兴趣，我目前的技术学习围绕以下几个方向展开： ### 2.1 编程基础 以 Python 作为入门语言，逐步掌握编程的基本逻辑和数据结构。在此基础上，根据课程安排进一步学习 Java 或 C# 等面向对象语言。 ### 2.2 数据库与数据管理 数据库是信管专业的核心课程之一。目前正在系统学习 SQL 语言和关系数据库设计原理，后续计划了解 NoSQL 数据库的基本概念。 ### 2.3 信息系统开发 从需求分析、系统设计到编码实现和测试部署，了解完整的系统开发生命周期。本次个人博客的搭建，就是对这一流程的初步实践。 ### 2.4 数据分析基础 掌握基本的数据处理和分析方法，包括数据清洗、统计分析和可视化呈现，为后续的数据驱动决策打下基础。 ## 三、本次博客搭建实践 本次个人博客的搭建基于以下技术栈： | 项目 | 选型 | |------|------| | 静态站点生成 | VitePress | | 内容格式 | Markdown | | 版本控制 | Git + GitHub | | 自动化部署 | GitHub Actions | | 网站托管 | GitHub Pages | **核心流程**：在本地使用 Markdown 编写文章 → 推送到 GitHub 仓库 → GitHub Actions 自动构建 → 部署到 GitHub Pages。 通过这次实践，我对"信息系统"的整个运作流程有了更直观的感受：一个系统从设计、开发到部署上线的完整过程，正是信管专业所学知识的具象化体现。 ## 四、阶段总结与展望 ### 4.1 对信管专业的理解 经过一段时间的学习和实践，我认为信管专业的核心价值在于"连接"。它培养的不是纯粹的程序员，也不是纯粹的管理者，而是能够在技术与业务之间搭建桥梁的人。 在企业的信息化建设中，既需要有人懂得技术的边界，也需要有人理解业务的需求。能够同时站在两个角度思考问题，是信管专业毕业生的独特优势。 ### 4.2 后续学习计划 1. **深入专业课程**：认真学好系统分析与设计、管理信息系统等核心课程； 2. **加强技术实践**：通过课程项目和开源参与，积累实际开发经验； 3. **拓展跨学科视野**：关注信息技术在不同管理场景中的应用案例。 ### 4.3 博客维护计划 本博客将作为个人学习过程的记录平台，后续不定期更新以下内容： - 课程学习笔记与知识总结； - 项目开发过程中的技术记录； - 对信息管理领域的观察与思考。 ## 五、结语 技术可以学习，工具可以熟练，但真正重要的是理解"为什么做"以及"为谁做"。信息管理与信息系统这个专业教会我的，不仅是如何使用技术，更是如何用技术的思维去解决管理中的实际问题。 路还很长，我会继续走在信管的这条路上，踏实地学下去。 --- *吴睿轩* *2026年7月14日*
